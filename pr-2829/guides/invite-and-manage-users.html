Invite users and manage their access

Note

This documentation is relevant to the new IBM Quantum® Platform. If you need the previous version, return to the IBM Quantum Platform Classic documentation.

Before you begin

Create your account's instances before you invite users. For every instance you create through IBM Quantum Platform, an associated access group is automatically generated (unless you decline the automatically generated access group when you create the instance). When you invite users to an instance using IBM Quantum Platform, you can assign them to a specific instance's access group as part of the invitation process.

Invite users to your organization's account

Users with IBM Cloud accounts

If a user has a previously existing IBM Cloud account, follow the instructions in the Manage IBM Cloud users section.

  1. First, ensure you have the correct account and region selected in the account and region switchers in the top header.

  2. Select Access management in the left-hand menu.

  3. Select the Users tab and click "Invite user +".

  4. You are prompted to enter up to 100 email addresses, separated by commas, spaces, or line breaks.

    You can optionally assign the invited user to an access group. To create an access group, see the Create and manage access groups guide.

  5. The user will receive an invitation to create an IBM Cloud account and join the instance.

Manage a user's access groups

To change a user's access group or to add a user to a different access group, go to Access management in the left-hand menu. Click the three-dots menu at the end of the row for that user and select Manage access. On the modal that opens, select the access groups the user should belong to. A user can belong to multiple access groups.

Remove a user

To remove a user from an instance, go to Access management in the left-hand menu. Click the three-dots menu at the end of the row for that user, and select Delete. You will be prompted to confirm the deletion.

Note

If a user is removed from an instance, their workloads and the associated results are no longer visible to them. Other users in the same instance can still access the workload data and results.
