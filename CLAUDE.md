# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Writing style

This repository follows **IBM Style** guidelines for all documentation content. When editing or creating documentation, adhere to these rules:

### Voice and tone

- Use **second-person** (you) instead of first-person (we, I)
  - *Exception:* Tutorials and learning material can use "we" and "our," but never "I"
- Use **active voice** over passive voice
  - Good: "The Estimator primitive returns a list of values"
  - Bad: "A list of values is returned by the Estimator primitive"
- Avoid future tense; don't use "will" unless necessary

### Punctuation and formatting

- Use **Oxford comma** in all lists (required)
  - Good: "X, Y, and Z"
  - Bad: "X, Y and Z"
- Use **&mdash;** for em-dashes (HTML entity only, not Unicode or hyphens)
- Use **sentence case** for headings, not Title Case
- Do not use Latin abbreviations (i.e., e.g., etc., vs.); use alternatives instead (Use "that is," "for example," "and so on," or "versus")

### Word choice

- No "please"
- No "may" — use "can" or "might" depending on context
- Avoid "which" without a comma; use "that" without a comma when restrictive
  - Good: "The value that changed" or "The value, which changed, is important"
- Avoid anthropomorphizing inanimate objects
  - Bad: "The software lets you..." or "The product enables you to..."
  - Good: "You can use the software to..." or "With this product, users can..."
- Use simple, direct words: "use" not "utilize," "run" not "execute"
- Use American English spelling

### Headers and titles

- Avoid -ing language in titles; use direct action-oriented language
  - Good: "Test circuits"
  - Bad: "Testing circuits"

### Dates

- Format dates as: **DD Monthname YYYY** (no comma after month)
  - Example: "13 February 2024"
