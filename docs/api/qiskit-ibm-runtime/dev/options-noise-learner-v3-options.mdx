  ### layer\_pair\_depths

  <Attribute id="qiskit_ibm_runtime.options.NoiseLearnerV3Options.layer_pair_depths" attributeTypeHint="list[int]" attributeValue="(0, 1, 2, 4, 16, 32)">
    The circuit depths (measured in number of pairs) to use in Pauli Lindblad experiments.

    Pairs are used as the unit because we exploit the order-2 nature of our entangling gates in the noise learning implementation. For example, a value of `3` corresponds to 6 repetitions of the layer of interest.

    <Admonition title="Note" type="note">
      This field is ignored by TREX experiments.
    </Admonition>
  </Attribute>

  ### num\_randomizations

  <Attribute id="qiskit_ibm_runtime.options.NoiseLearnerV3Options.num_randomizations" attributeTypeHint="int" attributeValue="32">
    The number of random circuits to use per learning circuit configuration.

    For TREX experiments, a configuration is a measurement basis.

    For Pauli Lindblad experiments, a configuration is a measurement basis and depth setting. For example, if your experiment has six depths, then setting this value to `32` will result in a total of `32 * 9 * 6` circuits that need to be executed (where `9` is the number of circuits that need to be implemented to measure all the required observables, see the note in the docstring for [`NoiseLearnerOptions`](options-noise-learner-options "qiskit_ibm_runtime.options.NoiseLearnerOptions") for mode details), at [`shots_per_randomization`](#qiskit_ibm_runtime.options.NoiseLearnerV3Options.shots_per_randomization "qiskit_ibm_runtime.options.NoiseLearnerV3Options.shots_per_randomization") each.
  </Attribute>

  ### rep\_delay

  <Attribute id="qiskit_ibm_runtime.options.NoiseLearnerV3Options.rep_delay" attributeTypeHint="float | None" attributeValue="None">
    The repetition delay.

    This is the delay between the end of one circuit and the start of the next within a shot loop. This is only supported on backends that have `backend.dynamic_reprate_enabled=True`. It must be from the range supplied by `backend.rep_delay_range`. When this value is `None`, the default value `backend.default_rep_delay` is used.
  </Attribute>
