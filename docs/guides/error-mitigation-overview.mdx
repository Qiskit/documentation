# Overview of noise management techniques

When executing quantum workloads, there are multiple ways to reduce the impact of noise. The open-source Qiskit addons provide error mitigation and suppression techniques that integrate directly into your development workflow, while Qiskit Runtime applies advanced error mitigation strategies automatically when jobs are submitted for execution. This page indexes all available tools and features across both options to help you choose the right approach to manage noise when building quantum workloads.

## Error mitigation for expectation value estimation

    <Card
      title="Twirled readout error extinction (TREX)"
      description="Error mitigation tool within Qiskit Runtime which mitigate the effects of measurement errors by randomly substituting them with a twirled measurement sequence."

      linkText="Learn more"
    />
    <Card
      title="Zero-noise extrapolation (ZNE)"
      description="Error mitigation technique that computes the expectation value at different noise levels, and then estimates the ideal result by extrapolating the noisy expectation value results to the zero-noise limit."
      href="/docs/guides/error-mitigation-and-suppression-techniques#zero-noise-extrapolation-zne"

    />
    <Card
      title="Probabilistic error amplification (PEA)"

      href="/docs/guides/error-mitigation-and-suppression-techniques#probabilistic-error-amplification-pea"
      analyticsName="Documentation page: PEA"
      linkText="Learn more"
    />
    <Card
      title="Probabilistic error cancellation (PEC)"
      description="Returns an unbiased estimate of the expectation value, at the expense of greater overhead than other techniques such as ZNE. It extrapolates output of the ideal circuit by executing different noisy circuit instances."
      href="/docs/guides/error-mitigation-and-suppression-techniques#probabilistic-error-cancellation-pec"
